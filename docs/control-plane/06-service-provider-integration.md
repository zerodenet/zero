# 机场（服务提供商）集成边界

本文档明确 Zero 内核与上层机场管理系统的边界，以及内核需要预留的集成点。

## 核心原则

**Zero 是代理内核，不是完整的机场管理系统。**

内核只提供原子能力，不实现业务逻辑：

| 内核提供 | 上层机场系统实现 |
| --- | --- |
| 流量转发 | 用户数据库 |
| 连接认证 | 套餐/计费/余额 |
| 流量统计 | Web 管理面板 |
| 事件导出 | 节点集群管理 |
| 配置热重载 | 订阅系统 |
| 节点健康探测 | 支付/工单/通知 |

---

## 内核需要预留的集成点

### 1. 多租户归因机制

**当前状态**: 已在 `flow.completed` 事件中预留 `principal_key` 字段

**需求**:
- 每个入站连接可关联一个租户标识
- 所有流量统计按租户聚合
- 事件中携带租户标识供上层计费

**实现方式**:
```rust
// 入站配置中预留
{
  "inbounds": [
    {
      "tag": "vmess-public",
      "type": "vmess",
      "listen": "0.0.0.0:443",
      "users": [
        {
          "id": "user-uuid-123",
          "principal_key": "user-001"  // 归因标识，写入事件
        }
      ]
    }
  ]
}
```

---

### 2. 细粒度事件 Sink

**需求**: 机场需要实时收集所有连接的流量数据用于计费

**内核提供**:
- 可配置的 `flow.completed` 事件导出
- 支持批量 HTTP Webhook 投递
- 支持本地 JSONL 文件落盘
- 支持事件过滤（只导出需要的字段）

**事件格式**:
```json
{
  "event_type": "flow.completed",
  "principal_key": "user-001",
  "inbound_tag": "vmess-public",
  "remote_addr": "1.2.3.4:5678",
  "bytes_up": 1048576,
  "bytes_down": 8388608,
  "duration_ms": 12345,
  "started_at_unix_ms": 1713500000000,
  "ended_at_unix_ms": 1713500012345
}
```

上层系统负责：
- 接收并存储事件
- 按 `principal_key` 聚合统计
- 计算费用/扣减余额
- 异常流量检测

---

### 3. 动态用户管理（热重载）

**需求**: 不需要重启节点即可添加/删除用户

**内核提供**:
- 配置热重载 API (`config.apply`)
- 重载时不中断已有连接
- 原子性配置切换

**上层系统负责**:
- 用户 CRUD 管理
- 生成完整配置
- 调用内核热重载接口

---

### 4. 节点健康与状态 API

**需求**: 机场控制面板需要监控大量节点的状态

**内核提供**:
- `GET /api/v1/health` 存活检查
- `GET /api/v1/stats` 流量/连接统计
- `GET /api/v1/runtime` 完整运行时状态
- `policy.probe.completed` 出站探测结果

**上层系统负责**:
- 多节点状态聚合
- 告警通知
- 负载均衡调度
- 节点上下线管理

---

### 5. 出站策略动态切换

**需求**: 根据用户等级、地区、时间等动态选择出站

**内核提供**:
- `policies.select` 命令切换当前出站
- 支持多种 `selector` 和 `urltest` 策略组

**上层系统负责**:
- 用户等级与出站映射规则
- 地区/时间调度逻辑
- 动态调用策略切换 API

---

### 6. 速率限制（预留）

**需求**: 按用户配置速率限制

**内核预留**:
- 可基于 `principal_key` 的流量控制钩子
- 具体限流逻辑由上层实现或插件提供

---

## 明确不进入内核的功能

### ❌ 用户认证数据库
- 不内置 SQLite/PostgreSQL 用户存储
- 不实现注册/登录/密码重置
- 上层系统管理用户，生成配置推送给内核

### ❌ 计费与支付
- 不内置套餐定义
- 不实现计费逻辑
- 不对接支付网关
- 内核只导出原始流量数据

### ❌ Web 管理面板
- 不提供用户可见的 UI
- 只提供 API 供上层面板调用

### ❌ 订阅系统
- 不实现订阅链接生成
- 不处理客户端订阅请求
- 上层系统负责订阅分发

### ❌ 多节点集群管理
- 不实现节点间通信
- 不实现配置同步
- 上层控制面统一管理多节点

## 扩展机制

### 插件化（未来）
对于机场可能需要的定制能力，通过插件机制提供：
- 自定义认证逻辑
- 流量整形/限速
- 特殊的事件处理

插件作为独立 crate，不进入内核。

### API 版本兼容
- 内核 API 遵循语义化版本
- 废弃 API 提前 2 个版本标记
- 上层系统可以平滑升级

---

## 总结

**内核做什么（原子能力）：**
✅ 高性能流量转发
✅ 标准入站协议认证
✅ 按用户归因的流量统计
✅ 归一化事件导出
✅ 配置热重载
✅ 策略组切换
✅ 节点健康信息

**内核不做什么（业务逻辑）：**
❌ 用户数据库和认证系统
❌ 套餐、计费、支付
❌ Web 管理面板 UI
❌ 多节点集群管理
❌ 订阅分发系统

这种边界设计保证了 Zero 内核的专注、轻量和高性能，同时为上层机场系统提供足够灵活的集成能力。
